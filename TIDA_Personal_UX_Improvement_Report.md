# TIDA Personal — MENTA参考UI/UX改善提案レポート

**作成日：** 2026年4月13日  
**調査対象：** [MENTA（menta.work）](https://menta.work/) — トップ・プラン一覧・プラン詳細・Aboutページ  
**改善対象：** TIDA Personal（パーソナルトレーナーマッチングサイト）5ページ

## 1. MENTAのUI/UX詳細分析

### 1-1. トップページ構成（セクション順）

| # | セクション | 内容・特徴 |
|---|---|---|
| 1 | グローバルナビ | ロゴ左／「カテゴリから探す」ドロップダウン中央左／「ログイン」「無料登録」右上。常時スティッキー表示 |
| 2 | ヒーローセクション | MENTOR→MENTEEのスクロールアニメーション文字＋「オンラインでふたりをマッチング！」＋「会員登録(無料)」メインCTA |
| 3 | 選ばれる3つの理由 | ①メンター数（7,400名）②個別指導 ③リーズナブル（平均1.4万円）— 数値による具体的な価値証明 |
| 4 | キャリア別導線 | 6パターン（エンジニア/デザイナー/マーケター/副業/起業/キャリアアップ）でユーザーを分岐させる |
| 5 | カテゴリ一覧 | 65以上のスキルをタブ切り替え表示。カテゴリ→スキルの2階層ナビ |
| 6 | 人気メンター一覧 | カテゴリ別（プログラミング/デザイン/ビジネス/動画）に4〜5名ずつカード表示 |
| 7 | 他社比較表 | MENTA vs スクール vs 学習サービスの3社比較。料金・メンター数・カスタマイズ・質問・コミュニケーション・期間を表で比較 |
| 8 | ユーザーボイス | Twitter埋め込み（実名＋アイコン付き）10件以上。リアルな口コミ感を演出 |
| 9 | メディア掲載 | 各メディアロゴをグレースケールで並べ、信頼性を担保 |
| 10 | 最終CTA | 「まずは無料登録からはじめよう」＋大きなボタン＋メンター向け別導線 |

### 1-2. ナビゲーション設計

- **メガメニュー型ドロップダウン：** 「カテゴリから探す」クリックで全カテゴリ（65以上）が2列グリッドで展開
- **デュアルCTA：** 「ログイン」（テキストリンク）と「無料登録」（塗りつぶしボタン）を並置し、新規登録を優先誘導
- **ページ内スクロール補助：** プラン詳細ページでは「プラン説明 / コース一覧 / 投稿レビュー」のアンカーリンクバーを設置
- **サブナビ：** スマホ向けには「メンターを探す」「メンターを募集」の2択シンプル構成

### 1-3. CTAボタン設計

| 場面 | 文言 | 特徴 |
|---|---|---|
| ナビ常設 | 「無料登録」 | 「無料」を明示。緑系アクセントカラー |
| ヒーロー | 「会員登録(無料)」 | 括弧で「無料」を再強調 |
| プラン詳細・即申込 | 「いますぐお願いしたい」 | 行動の即時性を強調 |
| プラン詳細・検討中 | 「まずは事前に相談だけしたい」 | ハードルを下げる中間CTA |
| プラン詳細・興味段階 | 「少しだけ興味があります」 | 最低コミットのマイクロCTA |
| フッター周辺 | 「今すぐはじめる」 | シンプルな行動喚起 |
| 問い合わせ前 | 「事前に質問する（無料）」 | 無料で相談できることを強調 |

**ポイント：** 単一CTAではなく、コミットメントレベル別に3〜4段階のCTAを用意し、迷っているユーザーを取りこぼさない設計。

### 1-4. プラン一覧ページの表示方式

```
[サイドバー フィルター]         [メインコンテンツ]
- 学習方法（チャット/ビデオ）    並び替え：おすすめ / レビュー件数順 / 新着
- メンター情報（本人確認済等）   ┌─────────────────────────────────┐
- 料金（6段階スライド）         │ [人気] バッジ                    │
- 対応時間帯（5区分）           │ プラン名（長め・絵文字入り）       │
- 返信スピード                   │ タグ: PHP Python WordPress +2    │
                                 │ 自己紹介の冒頭2〜3行             │
                                 │ ★4.7  メンター名（実績を含む）   │
                                 └─────────────────────────────────┘
```

**カードの情報密度：** プラン名・スキルタグ・自己紹介冒頭・評価・メンター名・累計契約数・バッジ（人気/NEW/おすすめ）を1カードに集約。情報量が多いが階層が明確。

### 1-5. プラン詳細ページのレイアウト

```
[左カラム（メイン）]                      [右サイドバー]
- パンくずリスト                           - 3段階CTA（即申込/相談/興味）
- タグ（スキル）                           - メンタープロフィール
- プランタイトル                             ・名前
- できることリスト（チェックマーク付き）      ・契約実績数
- プランの特徴（長文テキスト）               ・評価スコア（★）
- コースメニュー一覧                         ・本人確認済バッジ
  （おすすめ強調 + 複数プライシング）         ・NDAバッジ
- レビュー一覧（★・日付・コメント）        - スケジュール（返信速度）
  ※未登録ユーザーは途中で隠す              - 「事前に質問する（無料）」
```

### 1-6. 信頼感を高めるUI要素

1. **定量実績の多重表示：** 「約100,000人が利用」「7,400名のメンター」「平均契約価格帯1.4万円」「累計契約5000突破🔥」など、数字を随所に散りばめる
2. **ユーザー評価（星評価）：** 4.7、4.9、5.0 など小数点付き評価。レビュー件数も括弧で明示（例：4.9(61件)）
3. **バッジ制度：** 「人気」「NEW」「おすすめ」「実績あり」「本人確認済」「NDA合意済」「経歴登録済」
4. **Twitter口コミ埋め込み：** アイコン・日付・本名付きで10件以上。ステルスマーケティングではなく公開ツイートを活用
5. **メディア掲載ロゴ：** 著名メディアへの掲載をロゴで視覚的に証明
6. **比較表：** 他社スクール（30万円〜）との対比で「リーズナブル」を強調

## 2. TIDA Personal との差分・改善提案

### 2-1. tida_personal_top.html（ランディングページ）

#### 差分・課題

| 項目 | MENTA | TIDA Personal（現状） | 差分 |
|---|---|---|---|
| ヒーローキャッチコピー | 「オンラインでふたりをマッチング！」＋会員登録CTA | 要確認 | 数値を伴う価値証明が必要 |
| 信頼数値の表示 | トレーナー数・利用者数・平均価格 | 不明 | 具体的数字が不足していると推測 |
| 選ばれる理由 | 3ポイント＋数値根拠 | 不明 | 数字なしの抽象的メリットの可能性 |
| ユーザーボイス | Twitter埋め込み10件以上 | 不明 | SNS口コミ活用が弱い可能性 |
| 比較セクション | 他社スクール比較表 | なし（推測） | 競合優位性の訴求不足 |
| ユーザー分岐CTA | 6キャリアパス | なし（推測） | ユーザー種別の絞り込みが弱い |

#### 改善提案

**① ヒーローの数値化**
- 現在：「あなたにぴったりのトレーナーを見つけよう」（抽象的）
- 改善案：「**〇〇名のトレーナー**から、あなたに合った1人をオンラインでマッチング」（登録済みトレーナー数を明示）

**② 価値証明3点セット**
```
┌─────────────────────────────────────────────────────────────┐
│ ① XXX名のトレーナーから選べる   ② マンツーマン指導    ③ 月X,XXX円〜 │
│   目標別・得意分野で絞り込み     体型・目標に合わせた     大手ジムの  │
│   可能                           オーダーメイドメニュー   1/3の価格  │
└─────────────────────────────────────────────────────────────┘
```

**③ ゴール別分岐CTA（MENTAのキャリア別導線を応用）**
- 「ダイエットしたい」「筋肉をつけたい」「競技向けトレーニング」「産後ボディメイク」「健康維持したい」「姿勢改善したい」の6〜8パターンでユーザーをセグメント化し、それぞれ最適なトレーナー一覧へ直結する動線を設置

**④ SNS口コミセクションの追加**
- Twitterやインスタグラムの実際の投稿を埋め込み（ #TIDAPersonal などのハッシュタグを活用）
- テキスト証言だけでなく「Before/After写真付き」の口コミを優先表示

**⑤ 比較表の設置**
```
               TIDA Personal   大手ジム     オンライン動画
月額費用        X,XXX円〜       XX,XXX円〜   X,XXX円
指導スタイル    ◎ マンツーマン  △ 対面グループ × 動画のみ
進捗管理        ◎ 毎回記録      × なし        × なし
食事指導        ◎ 対応可        △ 別料金      × なし
```

### 2-2. tida_personal_trainers.html（トレーナー一覧）

#### 差分・課題

| 項目 | MENTA | TIDA Personal（現状推測） |
|---|---|---|
| フィルター項目 | 学習方法・料金・時間帯・返信スピード・資格確認 | 専門分野・価格帯程度と推測 |
| カードバッジ | 人気/NEW/おすすめ/実績あり | バッジなし（推測） |
| 並び替え | おすすめ/レビュー件数/新着 | ソート機能が弱い（推測） |
| 累計実績表示 | 「累計契約5000突破🔥」 | 指導人数の表示が薄い可能性 |

#### 改善提案

**① フィルターの充実**
```
[フィルターサイドバー]
- 専門ジャンル（ダイエット/筋トレ/産後/競技/姿勢矯正等）
- トレーニング形式（オンライン/対面/両対応）
- 料金（〜5,000円 / 〜10,000円 / 〜20,000円 / それ以上）
- 対応時間帯（早朝/昼間/夜間/週末）
- 資格確認済（NSCA/NESTA/NESTAなど）
- 性別（男性/女性/指定なし）
- 指導経験（初心者向け/上級者向け）
```

**② カードへの「信頼バッジ」追加**
- `[資格認定済]` `[人気]` `[NEW]` `[指導実績XXX名]` バッジをカード左上に配置
- 評価スコアは4.7のように小数点付きで表示（「普通の星5」より具体的に見える）

**③ カード情報密度の最適化**
```
┌────────────────────────────────────────────────┐
│ [人気] [資格認定済]                             │
│ 🏃 ダイエット特化 / ビデオ通話対応              │
│ トレーナー名（自己紹介の最初の1行）              │
│ 専門: ダイエット  筋力アップ  姿勢矯正  +1      │
│ ★4.9  累計指導 230名  月額 ¥12,000〜           │
└────────────────────────────────────────────────┘
```

**④ 「トレーナーを募集する」機能の追加**
MENTAの「メンターを募集する」機能を応用。希望条件を書いて逆指名できる導線をカード一覧の下部や空き枠に設置。希望のトレーナーが見つからないユーザーの離脱を防ぐ。

### 2-3. tida_personal_plans.html（プラン一覧・MENTAスタイル）

#### 差分・課題

MENTAスタイルを参考に設計しているとのことだが、トレーニング文脈でのカスタマイズが必要。

#### 改善提案

**① プラン種別の明確化**
MENTAは「単発1,000円〜 / 月額3,000円〜」の2軸を明示。TIDA Personalも以下の軸で整理することを推奨：
- 「単発相談（1回）」 vs「月額継続（月X回）」vs「プログラム（期間固定）」
- 価格を最小値から表示（「¥X,000〜」）で心理的ハードルを下げる

**② プランカードへのビジュアル化**
テキストだけでなく、以下の要素を視覚化：
- 「月2回のビデオ通話」→アイコン付きで表現
- 「食事指導あり」→チェックアイコン
- 「おすすめプラン」→目立つ色のバッジ

**③ コースメニュー階層化（プラン詳細内）**
MENTAはプラン詳細内に複数コース（チャット/ビデオ/単発/月額）を表示し、価格帯の選択肢を提供。TIDA Personalも1トレーナーが複数プランを持てる構造にし、入口のハードルを下げる（例：「お試し1回 ¥3,000」→「月額コース ¥15,000」）。

### 2-4. tida_personal_trainer_detail.html（トレーナー詳細）

#### 差分・課題

| 項目 | MENTA（プラン詳細） | TIDA Personal |
|---|---|---|
| サイドバーCTA | 3段階（即申込/相談/興味） | 単一CTAのみ（推測） |
| レビュー表示 | 全件 + 未ログインは途中で隠す | レビューがない or 少ない可能性 |
| メンター信頼要素 | 契約実績数・本人確認・NDA・スキル一覧 | 資格証明が薄い可能性 |
| 返信スピード表示 | 「24時間以内」をプロフィールに明記 | 応答速度の明示なし（推測） |

#### 改善提案

**① 3段階CTAの実装**
```
[右サイドバー]
┌───────────────────────────────┐
│ ● いますぐ申し込む           │  ← 主CTA（塗りつぶし・目立つ色）
│ ○ まずは無料で相談する       │  ← 副CTA（アウトライン）
│ ○ 気になる（お気に入り）     │  ← マイクロCTA（テキストリンク）
└───────────────────────────────┘
```

これにより「まだ決めていないユーザー」の離脱を防ぐ。

**② 信頼バッジの充実**
- `[本人確認済]`：身分証確認済みを示すバッジ
- `[資格証明済]`：NSCAやNESTAなど保有資格の公式認定リンク付き
- `[秘密保持(NDA)]`：個人情報取り扱い同意をバッジで可視化
- `[応答 24時間以内]`：初回返信の目安を明記

**③ 実績を数字で可視化**
- 「累計指導 XXX名」「継続率 XX%」「★ X.X（XX件のレビュー）」をプロフィール冒頭に大きく表示
- Before/After 写真（匿名可）を掲載できる機能

**④ レビューのUI改善**
- MENTAのようにログイン前は一部のみ表示 → 登録促進
- 星の内訳（5★〇件 / 4★〇件）を棒グラフで表示
- 日付の新しい順がデフォルト

**⑤ 「このトレーナーの他のプラン」セクション**
プラン詳細ページ下部に同一トレーナーの別プランを表示（MENTAの「このメンターの他のプラン」に相当）。クロスセルの機会を設ける。

### 2-5. tida_personal_quiz.html（診断フロー）

MENTAにはない、TIDA Personalの独自の差別化ポイント。より強化すべきセクション。

#### 改善提案

**① 診断結果の具体化**
診断後の結果ページで「あなたにおすすめのトレーナー3名」をカード表示し、診断からトレーナー詳細への動線を直結させる。

**② プログレスバーの実装**
「3/6 ステップ」のようなプログレス表示で「あとどれくらいか」を可視化し、離脱率を下げる。

**③ 診断開始のCTAコピー**
- 現在（推測）：「診断スタート」
- 改善案：「**90秒**でぴったりのトレーナーが見つかる診断」（時間を明示してハードルを下げる）

**④ 診断完了後のリターゲティング**
結果をブックマーク・共有できる機能（URLパラメータ保持）。「後で見返す」行動を促す。

## 4. UXフロー改善まとめ

### 現状の推定フロー
```
TOP → トレーナー一覧 → トレーナー詳細 → 申し込み
 ↓
（迷ったらそのまま離脱）
```

### 改善後の推定フロー

## 6. まとめ

MENTAの最大の強みは **「数字による具体的な信頼証明」** と **「コミットメントレベル別のCTA設計」** にある。単発1,000円というエントリー価格と「まずは相談」という中間CTA、そして累計契約数・評価点・本人確認バッジという多層的な信頼証明が組み合わさることで、見込みユーザーの心理的ハードルを段階的に下げている。

TIDA Personalはパーソナルトレーニングという高関与・高単価のサービスであるため、MENTAの設計思想は非常に親和性が高い。特に以下の3点を最優先に実装することで、CVRの大幅改善が見込まれる：

1. **信頼数値の最大化**（H-2・H-3・H-4）— トレーナー数・実績数・資格を数字で見せる
2. **3段階CTAの導入**（H-1）— 「まずは相談だけ」の選択肢でハードルを下げる
3. **ゴール別の分岐動線**（M-3）— ターゲットユーザーを入口で分けて最適なトレーナーへ誘導する
